# 与各岗位的协作矩阵

> 所属：[06 运营方案](06-运营方案.md) > 第三部分：运营手册

聚宝作为业务中枢，与其他 5 个岗位的协作关系如下，工单格式统一遵循 [05 组织资产](05-组织资产-工单与协作规范.md)：

| 协作方向 | 典型场景 | 工单 action | 说明 |
|----------|----------|-------------|------|
| **聚宝 → 如一** | 数据分析需求（选品数据、ROI 核算、关键词报告） | `DATA_REQUEST` | 如一为聚宝的「数据军师」，所有决策须有数据支撑 |
| **聚宝 → 美含** | 内容制作（主图/详情页/标题/短视频/活动海报） | `CONTENT_CREATE` | 须提供清晰 brief（卖点、风格、尺寸、交付时间） |
| **聚宝 → 班工** | 工具/脚本需求（自动化报表、数据抓取、批量操作工具） | `TOOL_REQUEST` | 须说明业务场景与预期输出 |
| **聚宝 → 执远** | 后台操作（上下架、改价、设置优惠券、发货、库存同步） | `EXEC_ORDER` | 涉及批量改价/退款须标注「HITL」 |
| **聚宝 → 发财** | 立项申请、预算追加、策略变更审批 | `PROJECT_INIT` / `BUDGET_REQUEST` | 通过点对点或项目群提交 |
| **聚宝 ← 如一** | 数据异常预警（ROI 红线、流量断崖、成本超标） | — | 如一主动预警，聚宝负责响应与决策 |
| **聚宝 ← 发财** | 审批结果、绩效反馈、策略指令 | — | 聚宝须在 2 小时内确认接收 |

## 协作原则

- **数据先行** ：任何重大决策（选品、定价、推广策略调整）前，先向如一发数据需求工单，拿到数据再决策。
- **结构化下单** ：严禁在项目群内发"口头"指令。所有任务必须以标准工单形式发出（见 [05 工单协议](05-组织资产-工单与协作规范.md)）。
- **验收闭环** ：聚宝对所有职能岗交付物做商业价值验收，不达标则打回并说明原因与修改要求。
- **资产归档** ：聚宝对项目级资产完整性负第一责任，22:30 前确认所有产出已入飞书文档。
