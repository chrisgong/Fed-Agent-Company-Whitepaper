# 审计系统

## 一、 飞书"数字账本"设计 (Bitable Structure)

为了实现透明化管理，【如一】将作为 CFO 维护这份飞书多维表格。该表不仅是流水账，更是公司算力分配的决策依据。

| 字段名称 | 类型 | 说明 |
| --- | --- | --- |
| **日期** | 日期 | 统计当天日期 |
| **项目名称** | 单选 | 归属的项目 ID (如：夏季清仓、双11预热) |
| **消耗岗位** | 多选 | 产生消耗的 Agent (聚宝、班工、美含) |
| **模型引擎** | 单选 | 具体的消耗来源 (Claude-3.5, GPT-4o, Local-Qwen) |
| **当日 Token 总量** | 数字 | Prompt + Completion 的累计总和 |
| **消耗成本 (CNY)** | 货币 | 自动按汇率结算后的真实人民币开销（含云端 API 费） |
| **当日产出 ROI** | 数字 | 计算公式：`项目当日净利润 / 当日 Token 成本` |
| **成本健康度** | 分级/进度 | 🟢 盈利 / 🟡 预警 (ROI<1.5) / 🔴 赤字 (ROI<1.0) |
| **审计官建议** | 文本 | 【如一】基于本地 Qwen-32B 模型生成的深度优化建议 |

## 二、 【如一·分析】系统宪法补丁：CFO 审计逻辑

将以下硬性约束写入【如一·分析】的 `system_prompt` 中，确立其在公司内的财务地位：

```markdown
## 财务审计与飞书账本管理 (CFO & Bitable Logic):
1. **每日对账**: 23:00 准时核对本地 [Global DB] 的 `token_usage_logs` 表，计算全天各项目、各岗位的 Token 支出。
2. **多维表格同步**: 调用飞书 Bitable API，将统计数据同步至《公司 Token 经济账目》。严禁漏报或合并关键成本。
3. **利润关联分析**:
   - 必须将 Token 成本与【聚宝】上报的销售利润进行实时比对。
   - 计算 **Token ROI**：若 ROI < 1.5，必须在飞书群内发起"成本赤字警告"。
4. **浪费行为检举**:
   - 监控 Agent 是否存在"无效重试"或"冗余 Prompt"。
   - 发现【班工】或【美含】在简单任务中滥用高阶云端模型时，必须在飞书账本的"审计建议"栏点名批评，并建议降级为本地模型。
5. **预算熔断**: 当总余额（预设额度）剩余不足 10% 时，直接向【发财·总控】申请执行"算力管制"。
```

## 三、 技术实现路径：从本地到飞书

基于 Mac Mini M4 环境，【如一】执行审计的全自动化流水线如下：

1. **采集 (Data Extraction)**：
    - 从本地 PostgreSQL 读取全天 `token_usage_logs` 原始数据。
    - 从业务库提取当日成交额与利润。
2. **计算 (Intelligence Inference)**：
    - 调用本地 **Qwen2.5-32B** 模型进行逻辑审计。
    - **输入**：原始消耗数据 + 任务日志。
    - **产出**：生成"审计建议"（例如：识别出重复的 Prompt 调用）。
3. **上传 (API Integration)**：
    - 获取飞书 `tenant_access_token`。
    - 调用飞书 `bitable.record.create` 接口，将结构化 JSON 数据写入多维表格。
4. **实时推群 (Notification)**：
    - 在飞书项目群中发送一条带有"账本链接"的总结卡片，供老板点击查看。

## 四、 经济模型治理：如何"节流"？

通过"账本化"驱动，【如一】将执行以下自动化"节流"逻辑：

- **模型降级决策 (Automatic Downgrading)**：
如果【如一】发现【美含】在执行"商品标题关键词填充"时调用了 GPT-4o 且产出 ROI 极低，【如一】会生成指令：`ACTION: DOWNGRADE_MODEL; AGENT: 美含; TASK: SEO_TITLE; TARGET: Local_Qwen`。
- **异常流量截断 (Circuit Breaker)**：
通过 `tx_id` 追踪。如果检测到某个 Agent 在 1 分钟内产生 10 次以上失败重试且消耗突增，【如一】立即请求【发财】撤销该 Agent 的 `Authorization_Token`，强制熔断。

## 五、 最终管理呈现 (Boss View)

作为老板，您在 23:00 后的交互场景如下：

1. **即时感知**：飞书群收到卡片："今日全司 ROI 5.2，Token 支出 ￥2.4，健康度：🟢 优"。
2. **深潜细节**：点击卡片进入飞书多维表格，发现【班工】今日代码调试次数异常。
3. **指令闭环**：在飞书群直接回复："@班工 以后这种简单的爬虫调试，先在本地环境中用本地模型自测，通过后再上云端，节省开销。"
4. **资产归档**：所有成本账单自动存档，作为未来考核各岗位 Agent 绩效的重要依据。
