# 项目生命周期

> 所属：[06 运营方案](06-运营方案.md) > 第二部分

店铺的运营以**开店指令**启动、以**关店指令**终止，中间为持续运营循环。所有指令通过飞书点对点发送给发财。

## 一、开店（项目启动）

老板通过飞书点对点向发财发送 **开店指令**，并附带店铺信息：

**开店指令格式** ：

```text
[开店指令]
店铺名称: {店铺名}
平台: 淘宝 / 天猫
店铺链接: {URL}
千牛账号: {账号}（登录态由人工完成首次登录）
1688 账号: {账号}
运营模式: 蓝海词杂货铺（全品类，不限品类方向）
备注: {其他需要说明的信息}
```

> **运营模式说明** ：当前阶段采用「蓝海词杂货铺」模式——不限定品类方向，所有蓝海词均可作为选品入口，最大化词池利用率与测款效率。后续若数据验证出高利润品类，可另开垂直店铺承接。

发财收到开店指令后执行以下动作：

| 步骤 | 执行者 | 动作 |
|------|--------|------|
| 1 | 发财 | 确认指令完整性，若信息缺失则追问补齐 |
| 2 | 发财 | 创建该店铺的**飞书项目群**，拉入所有相关 Agent（聚宝、如一、美含、班工、执远） |
| 3 | 发财 | 在项目群内发布《店铺启动通知》：含店铺基本信息、各岗位职责分工、初始目标 |
| 4 | 班工 | 配置执远的 Playwright 环境：千牛登录态目录、1688 登录态目录、哈士奇插件加载 |
| 5 | 执远 | 人工完成千牛和 1688 的首次登录，执远验证登录态有效 |
| 6 | 聚宝 | 进入日常运营循环（见 [06c 日常巡检](06c-日常巡检.md)） |

## 二、持续运营（日常循环）

开店流程完成后，进入**持续运营状态**，各 Agent 按运营手册中的 SOP 每日循环执行：

```text
┌─────────────────────────────────────────────┐
│              每日运营循环                      │
│                                               │
│  09:00  早盘巡检（数据/库存/差评/选词）        │
│    ↓                                          │
│  10:00  蓝海词选品 → 1688 找货 → 评品立项      │
│    ↓                                          │
│  14:00  午盘跟踪（流量/推广/转化实时监控）      │
│    ↓                                          │
│  持续    张飞搬家铺货 → 素材优化 → 测款         │
│    ↓                                          │
│  20:00  晚盘收盘（全天汇总/次日计划/EOD 复盘） │
│    ↓                                          │
│  循环 → 次日 09:00                             │
└─────────────────────────────────────────────┘
```

发财在运营期间的职责：
- 接收聚宝提交的立项申请并审批
- 监督各 Agent 工单执行进度
- 汇总 EOD 复盘报告，向老板汇报关键异常
- 响应老板的临时指令（调整策略、暂停推广等）

## 三、关店（项目终止）

老板通过飞书点对点向发财发送 **关店指令** ：

```text
[关店指令]
店铺名称: {店铺名}
关店类型: 暂停运营 / 永久关闭
处置要求: {如"清仓后关闭""立即下架所有商品""保留店铺但停止推广"等}
```

发财收到关店指令后执行以下动作：

| 步骤 | 执行者 | 动作 |
|------|--------|------|
| 1 | 发财 | 在项目群内发布《关店通知》，明确关店类型与处置要求 |
| 2 | 聚宝 | 停止新品选品与上新，暂停所有推广计划 |
| 3 | 执远 | 根据处置要求执行：下架商品 / 设置清仓价 / 关闭优惠券等 |
| 4 | 如一 | 输出《店铺运营终期报告》：累计 GMV、ROI、利润、商品表现排名等 |
| 5 | 发财 | 汇总关店报告提交老板，确认后归档项目群（不解散，保留记录） |
| 6 | 班工 | 清理该店铺的登录态配置、释放相关资源 |

**暂停运营** vs **永久关闭** ：
- **暂停运营** ：保留店铺配置与登录态，Agent 停止日常巡检但不清理环境，随时可通过发送「恢复运营」指令重新激活
- **永久关闭** ：执行完处置要求后，彻底清理配置与资源，归档所有数据
