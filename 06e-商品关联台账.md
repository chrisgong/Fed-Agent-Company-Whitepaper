# 商品关联台账（1688 ↔ 淘宝）

> 所属：[06 运营方案](06-运营方案.md) > 第三部分：运营手册

一件代发模式的核心数据资产。每一款在售商品都必须维护一条「1688 货源 ↔ 淘宝商品」的关联记录，贯穿发货、退货、利润核算全链路。

## 台账结构

台账以**飞书多维表**为主表（全员可查），同步写入 **PostgreSQL**（执远自动化读取）。

| 字段 | 说明 | 示例 |
|------|------|------|
| 淘宝商品 ID | 淘宝商品链接或 item_id | 78912345 |
| 淘宝商品名称 | 店铺内展示标题 | 户外折叠椅便携… |
| 1688 货源链接 | 对应 1688 商品页 URL | https://detail.1688.com/... |
| 1688 供应商 | 供应商店铺名 | XX 户外用品厂 |
| SKU 映射表 | 淘宝规格 → 1688 规格的对照关系 | 白色 L码 → 白 大号 |
| 拿货价 | 1688 单件成本（含运费） | ￥25.00 |
| 淘宝售价 | 当前店铺售价 | ￥59.90 |
| 毛利率 | 自动计算：(售价-拿货价-平台扣点-物流) / 售价 | 38% |
| 商品状态 | 在售 / 测款中 / 已下架 / 缺货暂停 | 在售 |
| 上架日期 | 铺货到淘宝的日期 | 2026-03-05 |
| 来源蓝海词 | 选品时使用的蓝海词 | 户外折叠椅 轻便 |
| 备注 | 供应商发货时效、品质备注等 | 通常 24h 内发货 |

## 台账维护规则

| 时机 | 执行者 | 动作 |
|------|--------|------|
| **新品上架** | 聚宝 | 张飞搬家铺货完成后，立即在台账新增记录，填写 1688 链接、SKU 映射、拿货价等 |
| **价格变动** | 聚宝 | 调整淘宝售价或 1688 供应商调价时，同步更新台账中的价格与毛利率 |
| **更换供应商** | 聚宝 | 原供应商缺货/品质问题需更换时，更新 1688 链接、SKU 映射、拿货价，保留旧供应商记录（标记历史） |
| **商品下架** | 聚宝 | 状态改为「已下架」，保留记录不删除（用于历史分析） |
| **供应商缺货** | 执远 | 发货时发现 1688 无货，将状态改为「缺货暂停」，通知聚宝 |
| **定期巡检** | 执远 | 每周 1 次自动校验所有在售商品的 1688 链接是否有效、价格是否变动 |

## 台账使用场景

| 场景 | 使用者 | 怎么用 |
|------|--------|--------|
| **发货** | 执远 | 收到淘宝订单 → 查台账找到 1688 链接 + SKU 映射 → 1688 下单 |
| **退货** | 执远 | 买家退货 → 查台账找到 1688 订单 → 向供应商发起退货 |
| **利润核算** | 如一 | 读取台账拿货价 + 淘宝售价，计算单品/整店利润率 |
| **选品复盘** | 聚宝 | 按蓝海词维度汇总：哪些词选出的商品利润率高、退货率低 |
| **供应商管理** | 聚宝 | 按供应商维度汇总：发货时效、缺货频次、品质退货率，淘汰差供应商 |
