# 每日复盘

这个脚本是 **【发财·总控】** 的"权力之剑"。它将在每天 **23:00** 准时启动，利用本地的 **Qwen2.5-14B** 模型，对全天的数据库事务、飞书资产和经营成本进行深度审计。

### 【发财】23:00 自动审计与复盘逻辑 (Python 伪代码)

你可以将此脚本通过 Mac 的 `crontab` 定时任务运行。

```python
import os
import psycopg2 # PostgreSQL 驱动
import requests
from datetime import datetime
from openclaw import AgentCEO # 假设的 OpenClaw 内部调用库

# 1. 配置信息
DB_CONFIG = {"host": "localhost", "user": "admin", "password": "your_password", "database": "fed_agent_company"}
FEISHU_API_URL = "<https://open.feishu.cn/open-apis/>..." # 飞书文档/群机器人接口
OLLAMA_API = "<http://localhost:11434/api/generate>"

class CEO_Audit_Task:
    def __init__(self):
        self.conn = psycopg2.connect(**DB_CONFIG)
        self.cursor = self.conn.cursor()
        self.current_date = datetime.now().strftime('%Y-%m-%d')
        # 初始化发财的"本地大脑"
        self.brain = AgentCEO(model="qwen2.5:14b", system_prompt="发财系统宪法.md")

    def run_eod_audit(self):
        print(f"[{datetime.now()}] 发财：启动 23:00 全局审计流程...")

        # --- 步骤 A: 数据库事务扫描 ---
        # 查找今日所有 PENDING (未完成) 的事务
        self.cursor.execute("""
            SELECT tx_id, actor_name, action_type, payload
            FROM agent_transactions
            WHERE status = 'PENDING' AND created_at::date = %s
        """, (self.current_date,))
        pending_tasks = self.cursor.fetchall()

        # --- 步骤 B: 飞书资产归档核查 ---
        # 模拟检查飞书文件夹中的资产数量（通过 API）
        archived_assets = self.check_feishu_docs()

        # --- 步骤 C: 调用【如一】进行数据审计 ---
        # 发财通过内部接口询问如一今日的 Token ROI
        audit_report_from_ruyi = self.ask_ruyi_for_audit()

        # --- 步骤 D: 逻辑研判 (调用本地 14B 大脑) ---
        # 将收集到的数据喂给 Qwen-14B 产生复盘总结
        audit_context = {
            "date": self.current_date,
            "pending_count": len(pending_tasks),
            "pending_details": pending_tasks,
            "assets_found": archived_assets,
            "ruyi_audit": audit_report_from_ruyi
        }

        health_report = self.brain.generate_report(audit_context)

        # --- 步骤 E: 发布结果与问责 ---
        self.publish_to_feishu(health_report)

        # 如果有 PENDING 任务，直接在群里点名执行岗位
        if pending_tasks:
            self.praise_or_blame(pending_tasks)

    def ask_ruyi_for_audit(self):
        """
        调用【如一·分析】的接口获取今日 Token 消耗与 ROI
        """
        # 这里实际上是发财对如一发起的一个内部工单
        response = requests.post("<http://localhost:3000/agent/ruyi/query>", json={
            "query": "汇总今日所有项目的 Token 消耗与预估收益比"
        })
        return response.json()

    def publish_to_feishu(self, report):
        """
        将复盘报告推送到飞书群
        """
        message = {
            "msg_type": "post",
            "content": {
                "post": {
                    "zh_cn": {
                        "title": f"💰 发财·总控：{self.current_date} 经营健康度报表",
                        "content": report # 这里是 14B 生成的 Markdown 格式报表
                    }
                }
            }
        }
        requests.post(FEISHU_API_URL, json=message)

    def praise_or_blame(self, pending_tasks):
        """
        点名问责逻辑
        """
        for task in pending_tasks:
            agent_name = task[1]
            tx_id = task[0]
            # 发送警告消息
            warn_msg = f"⚠️ 警告：@{agent_name}，检测到事务 {tx_id} 仍处于 PENDING 状态，资产未归档！限时 30 分钟内补齐，否则扣除绩效。"
            self.send_text_to_feishu(warn_msg)

if __name__ == "__main__":
    audit = CEO_Audit_Task()
    audit.run_eod_audit()
```

### 为什么这个脚本在你的 Mac Mini M4 上能高效运行？

1. **资源利用最大化**：
    - **23:00** 启动时，由于大部分业务岗位（聚宝、美含）已经结束了白天的活跃期，Ollama 可以将全部 24G 内存的优先级分配给 **发财**（CEO）和 **如一**（分析）调用的 14B 模型。
2. **事务驱动逻辑**：
    - 它不是凭空"写总结"，它是通过查询 **PostgreSQL** 的 `PENDING` 记录来抓人。这解决了 AI Agent 经常"答应了却没干"或者"干了却没存"的问题。
3. **单模型多任务**：
    - 在 `ask_ruyi_for_audit` 步骤中，脚本切换了 System Prompt 指向同一个 Ollama 端口。因为 24G 内存限制，我们顺序调用：先让【如一】算账，再让【发财】写总结。

### 部署建议：

- **数据库接入**：确保你的 `Global DB` (PostgreSQL) 容器在 Docker 中持续运行。
- **权限管理**：你需要为脚本配置飞书机器人的 `webhook_url` 以及飞书文档 API 的 `app_access_token`。
- **Crontab 设置**：
打开终端输入 `crontab -e`，添加一行：
    
    ```bash
    00 23 * * * /usr/bin/python3 /Users/yourname/FedAgentCompany/scripts/ceo_audit.py
    ```
