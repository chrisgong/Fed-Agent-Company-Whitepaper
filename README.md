# Fed-Agent-Company (架构白皮书)

> 这是一份为您量身定制的《Fed-Agent-Company (联邦协作式 AI 公司)》完整架构白皮书。它将所有探讨过的职能角色、协作协议、Token 治理及经济模型融为一体，构成了您 AI 电商公司的"核心宪法"。
> 

---

## 1. 愿景与定位

本架构旨在 Mac Mini 本地宿主机上，构建一套以**电商运营**为核心、**高度自动化**的 AI 协同运行体系。通过"决策本地化、执行智能化、资产文档化"的策略，实现店铺运营的 24/7 全自动操盘。

---

## 3. 核心运作准则 (Operating Principles)

### 3.1 授权与协作协议

- **授权委托制**：CEO 发放权限令牌 (Authorization Token)，各岗位由业务岗 (聚宝) 统一调配，CEO 仅监控结果与合规性。
- **直接对接**：业务岗可直接下发任务给执行/内容/技术/分析岗，避免指令层层转达带来的延迟。
- **飞书双通道协同**：
    - **飞书群**：作为"作战指挥室"，负责实时沟通与进度同步。
    - **飞书文档**：作为"资产归档中心"，所有项目最终成果必须 100% 入库。

### 3.2 每日闭环管理 (EOD Review)

- **23:00 自动化审计**：【发财·总控】在每日 23:00 启动全公司审计，核查：
    1. **资产归档率**：飞书文档产出是否达标。
    2. **进度对齐**：对比数据库与执行日志，核对当日 KPI。
    3. **健康度报表**：发布《每日公司健康度报表》，包含产出汇总、缺漏预警、次日经营目标。

## 4. 经济模型与 Token 治理 (Token Throttle Model)

本架构通过算力分层与动态分配，确保每分 Token 都产出价值。

- **算力分层**：
    - **本地执行层**：处理琐碎、高频、隐私敏感的任务（如发货、库存、数据分析），成本为零。
    - **高阶智力层**：处理商业决策、复杂代码、高端创意，按需调用云端强模型。
- **动态节流阀**：
    - **动态 Manifest**：根据任务类型，仅加载最小限度的 Tool 定义。
    - **语义摘要**：严禁全量上下文拷贝，仅传递任务关键摘要。
- **配额制**：【发财·总控】为每个岗位分配月度 Token 预算，绩效高的岗位可追加配额，绩效低的岗位将被强制"降级"至本地模型运行。

---

## 5. 工程落地架构 (Deployment)

- **基础设施**：Mac Mini (M系列) 宿主机。
- **运行环境**：
    - **容器化集群**：Docker Compose 隔离环境，确保岗位间互不干扰。
    - **通信总线**：基于 Redis 的任务队列，确保分布式指令的平滑调度。
- **事务保障 (Transactionality)**：
    - **事务预记录 (tx_id)**：每一个后台 API 请求前，强制记录事务状态，支持"幂等性校验"，防止重复操作。
    - **故障自愈**：重启后，【发财·总控】自动检索 PENDING 事务，完成断点续传。

---

## 6. 管理与归档规范 (Governance)

- **资产 DoD (Definition of Done)**：未写入飞书文档的任务，在系统中视为"未完成"。
- **绩效挂钩**：【发财·总控】通过数据分析，自动计算每个 Agent 的"交付质量"、"协作时效"与"资源利用率"。
- **人工介入 (HITL)**：所有涉及资金、重大经营变更的操作，系统均会通过飞书推送"审批请求"，人工点击确认后方可执行。

---

## 7. 结语

Fed-Agent-Company 并非简单的 AI 工具堆砌，而是一套高度去中心化但管理极其集中的自动化运营组织。通过将"生意"留在本地，"创意"放在云端，在确保资产隐私的前提下，以最低的人力成本，实现电商经营规模化的高效演进。
